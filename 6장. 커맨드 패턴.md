# 커맨드 패턴

**커맨드 패턴**을 사용하면 요청 내역을 객체로 캡슐화해서 객체를 서로 다른 요청 내역에 따라 매개변수화할 수 있습니다. 이러면 요청을 큐에 저장하거나 로그로 기억하거나 작업 취소 기능을 사용할수 있습니다.
→ **메소드 호출 캡슐화**

# 만능 IOT 리모컨 예제
- 상황
- 7개의 슬롯마다 각각 ON, OFF 버튼이 있음. 각 슬롯마다 연결된 제품과 상호작용 해야함.
- 슬롯에는 협력 업체의 제품이 연결되며, 제품에 대한 스펙으로 클래스를 제공 받았음. 하지만 각 클래스마다 인터페이스가 달라서 처리하기가 어려움.

<br/>

**커맨드 인터페이스 구현**
```java
 public interface Command {
     public void execute();
 }
```

<br/>

**커맨드 인터페이스 구현**
```java
 public class LightOnCommand implements Command {
     Light light;
     
     public LightOnCommand(Light light) {
         this.light = light;
     }
     
     public void execute() {
         light.on();
     }
 }
```
<br/>

**커맨드 객체 사용하기**
```java
public class SimpleRemoteControl {
    Command slot;
    public SimpleRemoteControl() {}
    
    public void setCommand(Command command) {
        slot = command;
    }
    public void buttonWasPressed() {
        slot.execute();
    }
}
```

<br/>

**리모컨 테스트 해보기**
```java
 public class RemoteControlTest {
     public static void main(String[] args) {
         SimpleRemoteControl remote = new SimpleRemoteControl();
         Light light = new Light();
         LIghtOnCommand lightOn = new LightOnCommand(light);
         
         remote.setCommand(lightOn);
         remote.buttonWasPressed();
     }
 }
```
<br/>

**NoCommand 객체**
```java
pulbic class NoCommand implements Command {
    public void execute() {}
}
```
- 일종의 널 객체(null object)
- 딱히 리헌할 객체도 없고 클라이언트가 null을 처리하지 않게 하고 싶을 때 활용하면 좋음
- 널 객체를 일종의 디자인 패턴으로 분류하기도 함
<br/>

**작업 취소 기능 추가**
```java
 public interface Command {
     public void execute();
     public void undo();
 }
```

```java
 public class LightOnCommand implements Command {
     Light light;
     
     public LightOnCommand(Light light) {
         this.light = light;
     }
     
     public void execute() {
         light.on();
     }
     
     public void undo() {
         light.off();
     }
 }
```

<br/>

**매크로 커맨드 사용하기**
```java
public class MacroCommand implements Command {
    Command[] commands;
    
    public MacroCommand(Command[] commands) {
        this.commands = commands;
    }
    
    public void execute() {
        for (int i = 0; i < commands.length; i++) {
            commands[i].execute();
        }
    }
}
```

1. 매크로에 넣을 커맨드 생성
```java
Light light = new Light("Living Room");
TV tv = new TV("Living Room");
Stereo stereo = new Stereo("Living Room");
Hottub hottub = new Hottub();

LightOnCommand lightOn = new LightOnCommand(light);
StereoOnCommand stereoOn = new StereoOnCommand(stereo);
TVOnCommand tvOn = new TVOnCommand(tv);
HottubOnCommand hottubOn = new HottubOnCommand(hottub);
```

2. 배열을 만들고 매크로 커맨드 객체 생성
```java
Command[] partyOn = { lightOn, stereoOn, tvOn, hottubOn};
Command[] partyOff = { lightOff, stereoOff, tvOff, hottubOff};

MacroCommand partyOnMacro = new MacroCommand(partyOn);
MacroCommand partyOffMacro = new MacroCommand(partyOff);
```

3. 객체를 버튼에 할당
```java
remoteContol.setCommand(0, partyOnMacro, partyOffMacro);
```

**커맨드 패턴 활용하기**
1. 리시버와 일련의 행동을 패키지로 묶어서 객체 형태로 전달할수 있음
    → 스케줄러나 스레드 풀, 작업 큐와 같은 다양한 작업에 적용할수 있음.
2. 로그 및 트랜잭션 시스템을 구현할 수 있음
    → 스프레드 시트와 같이 특정 체크 포인트 이후의 모든 행동을 로그에 기록하는 방식으로 복구 시스템을 구축할 수 있음.
    → 더 복잡한 애플리케이션에서는 일련의 작업에 트랜잭션을 활용해서 모든 작업이 완벽하게 처리되거나, 그렇지 않다면 아무것도 처리되지 않게 롤백처리 할수 있음.
